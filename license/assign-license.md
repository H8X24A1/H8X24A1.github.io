# Assign License

HXA.io supports two license types:

1. Per resource license
2. Per user license

The license type depends on the HXA.io application you want to use.

## Per Resource License

The following applications require a separate license for each resource:

- HXA Room Booking
- HXA Room Board
- HXA Door Connect
- HXA Environment Control
- HXA Room Service

This means that every room, device, or other supported resource must have its own assigned license.

### How to assign a per resource license

1. Open the HXA.io portal.
2. Navigate to the resource you want to configure.
3. Open the resource configuration.
4. Locate the license selection field.
5. Select the correct license for this resource.
6. Save the configuration.

The license must be attached directly inside the corresponding resource configuration.

## Per User License

The HXA Sparo App uses a per user license model.

This means the license is not assigned to a resource. Instead, it must be assigned to the user who will use the application.

### How to assign a per user license

1. Open the HXA.io portal.
2. Navigate to the user management area.
3. Open the list of users imported from Microsoft Entra ID.
4. Select the user who should receive the HXA Sparo license.
5. Choose the HXA Sparo license in the user settings.
6. Save the changes.

Users are typically imported from Microsoft Azure Active Directory, which is now called Microsoft Entra ID.

## Summary

- Per resource licenses are assigned inside the resource configuration.
- HXA Room Booking, HXA Room Board, HXA Door Connect, HXA Environment Control, and HXA Room Service use per resource licenses.
- Per user licenses are assigned inside the user configuration.
- HXA Sparo App uses a per user license for users imported from Microsoft Entra ID.
